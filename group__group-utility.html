#include <yenxo/type_name.hpp>

Get the name of a type.

The template TypeNameImpl has specializations for:

  • fundamental types;
  • any type compatible with either requirement: standard Container or MapContainer. The names are 'list' and 'map' respectively;
  • any enum defined via YENXO_DEFINE_ENUM. Define,
    YENXO_DEFINE_ENUM(E1, e1, e2);
    YENXO_DEFINE_ENUM
    auto YENXO_DEFINE_ENUM(...)
    Generates an enum and yenxo::EnumTraits for it.
    Test,
    static_assert(typeName(boost::hana::type_c<E1>) == "E1");
    REQUIRE(typeName(boost::hana::type_c<E1>) == "E1");
    yenxo::typeName
    constexpr auto typeName
    Get the name of a type.
    Definition: type_name.hpp:70
  • std::integral_constant. The function object should be enabled for underlying type;
  • std::optional. The function object should be enabled for underlying type;

There are few ways to enable the function object for user defined types:

  • specialize the TypeNameImpl;
    namespace {
    struct Def3 {};
    } // namespace
    namespace yenxo {
    template <>
    struct TypeNameImpl<Def3> {
    static constexpr std::string_view apply() noexcept {
    return "Def3";
    }
    };
  • provide auto typeNameImpl(T const*) function discoverable via ADL lookup;
    struct Def2 {
    friend constexpr std::string_view typeNameImpl(Def2 const*) noexcept {
    return "Def2";
    }
    };
  • provide auto typeName() static member.
    struct Def {
    constexpr static std::string_view typeName() noexcept {
    return "Def";
    }
    };
    Checks for user defined types:
    static_assert(typeName(boost::hana::type_c<Def>) == "Def");
    REQUIRE(typeName(boost::hana::type_c<Def>) == "Def");
    static_assert(typeName(boost::hana::type_c<Def2>) == "Def2");
    REQUIRE(typeName(boost::hana::type_c<Def2>) == "Def2");
    static_assert(typeName(boost::hana::type_c<Def3>) == "Def3");
    REQUIRE(typeName(boost::hana::type_c<Def3>) == "Def3");
