#include <yenxo/query_string.hpp>

Parse a query string.

Supports deep objects and arrays. The function, for overhead reasons, limits the resulting object dimensions:

  • object_property_count_limit=20;
  • object_depth_limit=20;
  • array_length_limit=20.

Examples

Scalar parameter:

REQUIRE(query_string("a=b") == R"({"a": "b"})"_j);
yenxo::query_string
Variant query_string(std::string const &str)
Parse a query string.

The _j string literal produces Variant from a JSON. The literal is introduced just to shorten examples and is not available in the library interface.

Array parameter by duplicating the parameter's name:

REQUIRE(query_string("a=b&a=c") == R"({"a": ["b", "c"]})"_j);

Array parameter by using empty square brackets:

REQUIRE(query_string("a[]=b") == R"({"a": ["b"]})"_j);

For Array parameters indexes can be specified (0 - first index):

REQUIRE(query_string("a[1]=b"_b) == R"({"a": [null, "b"]})"_j);

The array have size of max index plus one. The array have null-values for unspecified indexes.

Object parameter:

REQUIRE(query_string("a[b]=c") == R"({"a": {"b": "c"}})"_j);

Brackets can be (and recommended to be) percent-encoded:

REQUIRE(query_string("a%5bb%5d=c") == R"({"a": {"b": "c"}})"_j);

Array indexes ([2]) and object keys ([b]) can be composed in any way:

REQUIRE(query_string("a[b][1]=1&a[b]=2"_b)
== R"({"a": {"b": [null, "1", "2"]}})"_j);
REQUIRE(query_string("a[1][b]=1&a[1][c]=2"_b)
== R"({"a": [null, {"b": "1", "c": "2"}]})"_j);

Empty brackets can occur only on a key ending.

Error message may refer to ABNF grammar
